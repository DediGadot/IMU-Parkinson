Short version: iter29B is a plausible real signal, but I would treat it as **not yet canonical** until the chain-specific leakage/teacher-forcing checks pass and the LOOCV artifact is cleanly pre-registered.

1. **Mechanism**

The lift is mostly **item-covariance exploitation plus target decomposition**, not true multi-task sample-size gain.

`RegressorChain` is not joint optimization. It fits six scalar LGB models sequentially, so it does **not** make the effective N equal to `94 × 6 = 564`. Later heads get prior item predictions as extra covariates, so the chain can use “easy” axial items as a low-dimensional learned severity proxy for harder items.

That differs from:

- single-task T1 LGB: one noisy summed ordinal target, one feature-target map.
- independent per-item LGB: six maps, but no conditional correction among item residuals, so F53-style variance compounding can dominate.
- chain: six maps plus conditional item manifold constraints.

The biggest caveat is sklearn `RegressorChain(cv=None)`: during training, later heads see **true previous item residuals**, while at test time they see predicted previous residuals. That is not classic test leakage, but it is teacher-forcing mismatch and can overstate stability.

2. **LOOCV Survival**

Observed:

- iter29B mean 5-fold CCC: `0.7085`
- iter5-direct-T1 mean 5-fold CCC: `0.6572`
- mean Δ: `+0.0513`
- Δ seed SD from `[+0.047, +0.029, +0.078]`: about `0.025`

That seed SD already misses the repo’s T1 promotion discipline of 5 seeds with sum-level SD `<0.020`.

Given F53/F56/F58, I would not project the full `+0.05` into LOOCV. This is less doomed than the F56 19-feature meta-stack because it is a structured six-head target factorization, but it is still a k>2 label-side construction.

My forecast:

- expected LOOCV Δ: `+0.025` to `+0.035`
- expected LOOCV CCC: about `0.680` to `0.690`
- full `+0.05` survival: unlikely
- Δ positive but CI crossing zero: very plausible at N=94

If the `cv=None` teacher-forcing artifact is large, shrink that forecast to roughly `0` to `+0.015`.

3. **Leakage Checks**

Your listed pieces are not leaks if strictly fold-local:

- K=500 selector on train residual: OK if fit only inside outer train.
- item residual = item minus train-fold mean: OK if train mean only.
- random chain order: OK only if the seed/order ensemble was fixed before reading LOOCV.

Tests I would require:

- rerun with `RegressorChain(cv=inner_5fold)` or manual OOF prior-item covariates inside each outer train.
- independent per-item ablation with identical selector/base learner but no chain covariates.
- permute prior-item chain covariates inside train; lift should disappear.
- oracle diagnostic using true prior items at test to bound the maximum possible chain benefit.
- 100+ scrambled-label permutations, not only one, because N=94 CCC nulls can be noisy.
- SID-shuffle join and test-only canary as usual.

4. **Iter30 Priority**

Highest leverage is **30E: harden iter29B**, not broaden the search.

Freeze one formula: fixed item order, fixed seed ensemble, fixed K, fixed LGB params, fold-local means, and preferably OOF chain covariates. Then run null gates and LOOCV once.

30B is second, but only narrowly: medical-domain chain order or fixed equal-weight order ensemble. I would avoid CatBoost/XGB/blending until after the lockbox decision because that becomes iter11A-style adaptive variant search.

30A T3 all-18 chain is low priority. F53 and F56 say per-item T3 composition/meta-learning at this N is structurally fragile, and 18 heads greatly increase order/variance risk.

30C supervised contrastive KNN is low priority. At N=94 it is likely another target-leaky latent/retrieval variant unless deeply nested.

30D BMA is low priority for canonical use. F56/F58 already warn that same-sample mixers look better in screening than LOOCV.

5. **Lockbox Decision**

If, and only if, iter29B was pre-registered before LOOCV results/artifacts were observed, full null gates pass, and paired bootstrap versus **iter12 honest** excludes zero, then yes: it can replace `0.6550` as canonical T1.

The hazard is methodological, not just statistical: if LOOCV was launched or inspected before freezing the exact formula, seed ensemble, chain order, and comparison target, this becomes another adaptive-selection result. In that case I would report it as a strong post-hoc candidate, not the new canonical.
